关于对象

Javascript所有对象都继承自Object

所以一般对象有两种方法集：

  • Object本身的方法

  • 对象实例的方法

Object()

我们可以使用Object()来强制类型转换，将任意值转换为一个对象

我们可以分为三种情况

  • 空对象

    • Object()

    • Object(null)

    • Object(undefined)

  • 需要转换

    • Number

    • String

    • Boolean

  • 不需要转换

    • Array

    • Object

    • Function

不需要转换的可以有Object(...) === Object(...) //true

但是Object(undefined) === Object(undefined) //false

静态方法

Object.keys()
Object.getOwnPropertyNames()

这两者的区别在于Object.getOwnPropertyNames能够返回不可枚举的属性名

什么叫不可枚举的属性名呢?

类似一个数组包含了两个元素let a = [11,22],那么Object.keys返回的就是["0","1"],而另一个方法返回的是["0","1","length"]

这里面的length就是一个不可枚举的属性名

不过我们一般情况下是不希望将不可枚举的属性名抽取出来的，所以我们一般是用Object.keys()

Object.prototype.getOwnPropertyDescriptor

这个方法只能够用于自身的属性，作用是给出某个属性的详细信息(属性描述对象)

{
  value: 123,           //值
  writable: false,      //是否可写
  enumerable: true,     //是否可遍历(是否可以用Object.keys()遍历出来)
  configurable: false,  //控制了属性描述对象的可写性
  get: undefined,
  set: undefined
}

如果设置了get 或者 set，则不能再设置writable为true，以及不能设置value

Object.defineProperty(obj, prop, attr) && Object.defineProperties(obj, {prop: attr})

这是一个对属性比较详细的定义，一般来说我们就只是简单的赋值就好，这里具体到上面所说的属性描述对象

Object.prototype.propertyIsEnumerable(prop_name)

判断某个属性是否可遍历

实例方法

Object.prototype.valueOf()

返回当前对象的值，比如


let a = true

let oa = Object(a)

oa.valueOf()  //true

我们还可以重写这个方法


oa.valueOf = function() {
    //do something you want
};

如果我们重写了这个函数，但是又想调回原来的未重写前的方法，可以这样做

其实这样做，原理上是为这个Object增加了一个属性，就和我们自己定义一个Object的方法一样

Object.prototype.valueOf.call(oa)

这时候可以看到除了之前的value之外，还会有一个valueOf，可以看出valueOf已经作为其属性之一了

Object.prototype.toString()

用法同上

这里要区分不同的情况，因为默认情况下，以下的类型的toString是已经被默认重写了

  • Number

  • String

  • Boolean

  • Array

  • Function

  • Date

let obj = {}

obj.toString()  //"[object Object]"

同理，我们可以使用

Object.prototype.toString.call(oa)  //"[object Array]"

只要用这种方法即可忽视重写的方法

所以，toString可以用来判断类型

Object.prototype.toLocaleString()

和toString类似，只是提供不同版本

例如Date，可能toString()想要返回UTC时间，而toLocaleString想返回当地的时间

Object.prototype.hasOwnProperty()

判断Object是否拥有某个属性，注意输入参数是字符串

oa.hasOwnProperty("0")  //true

oa.hasOwnProperty("2")  //false

oa.hasOwnProperty("valueOf")

对象状态

Object.preventExtensions()  //不能增加属性
Object.isExtensible()

Object.seal()               //不能增加与删除属性
Object.isSealed()

Object.freeze()             //不能增加，删除和修改属性
Object.isFrozen()

要特别注意的是

我们所说的不能修改属性是和const性质是一样的，只是说明不能重新赋值

对于string，number之类的这种单一值的来说，就是常量的概念

但是对于数组来说，它本身是可以改变的，可以理解为：指向的内存区域不变，但是内存区域的内容可变
