阮一峰

数组的创建

假设我们创建一个数组

var arr = new Array(10)

其实这个数组是没有key的，但是我们却能够用下标访问，并且可以越界

Object.keys(arr)    //[]

arr[0] //undefined

arr[100] //undefined

一般来说不推荐这种创建数组的方式

var arr1 = [undefined,undefined]

Object.keys(arr1) //["0","1"]

静态方法

Array.isArray()

因为typeof对于数组智能返回一个object

所以我们可以用Array.isArray()来判断这个是否为数组

从这里我们可以看到，array也是一个对象

所以会有接下来的方法

valueOf() && toString()

valueOf 会把数据结构原封不懂的输出

typeof arr.valueOf()    //object

Array.isArray(arr.valueOf()) //true

说明它又是输出一个数组呀

那么toString就不同了，这个会把元素一个个拆分，然后用逗号隔开组成一个string

//数组
[1, 2, 3, Array(3), "aa"]

//toString结果
"1,2,3,4,5,6,aa"

push() pop() shift() unshift()

push和unshift都是往数组增加元素

pop和shift都是减少数组元素

push和pop都是在针对数组后面的元素

shift和unshift都是针对数组前面的元素

join()

这个会返回一个字符串，默认用逗号分隔，也可以传入分隔符

如果是undefined或者null会被转换为空字符串

concat

concat是数组的加法，可以将在原数组的基础上加上传入的参数，返回一个新的数组，原数组不变

要注意的是

如果数组成员包含对象，concat方法返回的是一个浅拷贝，也就是新数组拷贝的是对象的引用

因此当我们的对象改变的时候，它包含的对象也会相应改变

reverse

反转数组，这个是改变原来数组的

slice

提取数组的一部分，返回新数组，原数组不变

用整数表示从开头数起，负数表示从结尾数起

splice

arr.splice(start, count, addElement1, addElement2, ...);

在某个位置start开始算起(包括)，删除count个元素，再插入addElement

首先，start也适用于正负数

count可以为0，表示不删除

addElement可以为多个，表示插入多个

sort

sort默认是按照字典顺序排序，所以都会被转换成字符串再比较

并且该方法会改变原数组

当然，我们也可以自定义一个规则去排序(比如我们要排序Object，是按照Object内的某一个字段)

[
  { name: "张三", age: 30 },
  { name: "李四", age: 24 },
  { name: "王五", age: 28  }
].sort(function (o1, o2) {
  return o1.age - o2.age;
})

如果返回值大于0，则第一个元素在第二个元素后面，否则在前面

map

map接收一个函数作为参数

arr.map((e,i,arr) => {
    //do something
})

  • e: 数组的单个元素

  • i: 元素在数组的索引

  • arr: 数组本身

要注意的是

map会跳过空位，但是不会跳过undefined和null

看一个例子

var arr = Array(2)

arr.map(() => {return 'a'}) //啥也没有

arr = [undefined,null]

arr.map(() => {return 'a'}) // ["a","a"]

map除了传入一个函数参数意外，还能传入第二个参数作为函数的this变量，如

let var = [];
arr.map((e,i,arr) => {
    this.push(e);
},out);

forEach

forEach非常像map，它也对数组的所有成员依次执行参数函数

forEach也是接受三个函数参数

arr.forEach((e,i,arr) => {console.log(e)})

forEach也能接受第二个参数作为第一个参数的this变量

但是forEach不返回值，只是在函数内部进行操作，和map不同，map是会返回一个新的数组的

forEach和map都会跳过空位，但都不会跳过undefined和null

要注意的是

forEach并不能break，也就是不能中途退出，必定要遍历完所有的成员，所以如果要break,要使用for循环

filter

filter是数组里一个很重要的函数，并且经常和map连用

filter用于过滤数组成员，只有在callback函数里面返回true的成员才会被选中

被选中的成员组成一个新的数组，返回出来，对原数组无影响

[1,2,3,3,4,4,5].filter((e,i,arr) => {
    return e>3
});     //[4,4,5]

可以看出，filter的callback函数也可以接收3个参数

同理，filter也可以接收第二个参数作为callback函数的this变量

some && every

some是指：只要数组成员有一个满足callback函数的条件，就返回true

every是指：必须数组的每一个成员都要满足callback函数的条件，才返回true

从字面上可以理解为逻辑符号|| 和 &&

对于空数组

  • some总是返回false

  • every总是返回true

reduce && reduceRight

要谨记，reduce和reduceRight最终输出是一个值

reduce是从左到右，reduceRight从右到左

[1,2,3,4,5].reduce((a,b) => {
    return a+b
})  //15

上面的例子是典型的reduce求和，我们可以给reduce加上第二个参数，表示初始值

[1,2,3,4,5].reduce((a,b) => {
    return a+b
},20)   //35

reduce还能够找出字符串长度最大的成员

["a","aa","aaa"].reduce((a,b) => {
    return a.length > b.length ? a : b
},'') //"aaa"

indexOf && lastIndexOf

这是一个索引查找，如果查找结果不在数组内，则返回-1

我们还可以设置查找开始的位置

[1,2,3].indexOf(4,1)
//从索引为1的地方开始找4，因为4不在数组内，所以返回-1

[1,2,3].indexOf(1,1)
//因为从索引为1的地方开始找，而1又在0上，所以也返回-1

lastIndexOf只不过是调转了方向
