Linux命令笔记

2019年3月17日
| Linux
| 阅读

Command

这里只是笔者对Linux常用命令的笔记，方便自己查询

ifconfig 查看网络连接

lsof -i:8000 查看端口8000占用情况

kill -9 PID 把某个PID的进程结束了，kill还有许多flag，1:挂起， 2:中断 ，3:退出

mkdir -p /1/2/3/4 可以一次创建多层文件夹

pkill -f <name> 可以直接kill掉一个已知名字的进程

把脚本放进/etc/init.d/里面就可以自启动

chown user:group <file> 改变文件持有

磁盘

du -hsc <path> 查看路径占用磁盘大小

ncdu 更友好的查看磁盘占用情况的一个命令，需要安装

df -h查看磁盘占用情况

别名

alias <name>="<command>" 用别名调用命令(适用于长命令)，如果需要持久化的话，需要写入.bashrc

查找文件

find <path> -name "*.log" 寻找在路径里面结尾为.log的文件，同理，想要查找一类文件可以用这个命令

find <path> ! -name "*.log" 与上面刚好相反

find -mtime <number>这个命令还可以根据修改时间

匹配

grep -r "error" <path or file> 会打印出所有含有error的字段

查看CPU&内存占用情况

top 这个命令对界面不太友好

htop 这个对阅读更加友好

update 能够看到电脑持续开机时间

查看Linux发行版本

cat /proc/version or lsb_release -a

内存

虚拟内存和物理内存

内存的一项主要任务，就是存储进程的相关数据

进程并不能直接访问物理内存，进程中能访问的地址，只能是虚拟内存地址,这种内存管理方式，称为虚拟内存

每个进程都有自己的一套虚拟内存地址，进程的虚拟内存地址相互独立，所以两个进程空间可以有相同的虚拟内存地址

我们对进程的某个虚拟地址操作会被CPU翻译成对某个物理地址的操作

说到这里可以看出，应用程序对物理内存地址一无所知，无论是什么类型的语言，所表达出来的地址统统是虚拟内存地址

虚拟内存地址 ---> CPU ---> 物理内存地址

所以我们物理内存地址的掌控权在操作系统上，因此操作系统可以通过讲不同的进程映射到不同的内存区域来达到两个进程不往来，从而减少进程出错的可能性

内存共享

操作系统控制着物理内存，进程控制着虚拟内存

只要将操作系统的物理内存投射到不同的进程中的虚拟内存中，就可以达到无复制的数据共享(内存共享)

比如我们的内核，每个进程空间最初一部分的虚拟内存地址，都对应到物理内存中预留给内核的空间

内存分页

因为CPU在两种内存之间做了翻译者的角色，因此，这种模式会耗费计算机资源

最简单的翻译

把所有对应关系都记录在一个表中，这个表又必须放在内存当中，但是这种方式非常非常浪费资源，甚至会把所有物理内存占满，导致计算机崩溃

这种方法的映射关系是这样的

单个字节(虚拟内存)  ---> CPU ---> 单个字节(物理内存)

分页

Linux采用了分页的方式来记录对应关系

所谓分页，相对上面的最简单方法而言，分页将连续的地址(比如大小为4kb)组成一个页

所以在进程空间到物理内存之间的映射关系就变成了

页(虚拟内存) ---> CPU ---> 页(物理内存)

内存中的总页数就是内存字节数的1/4000

不过我们要清楚知道的是页中的内存地址是连续的

并且页的大小在虚拟内存和物理内存中是一样的，这样才能保证页内的数据一一对应

虽然是一一对应，那么我们怎么才能通过某一个虚拟内存地址来访问对应一个物理内存地址呢

通过偏移量， 虚拟地址包含了两个部分页编号和偏移量

比如我们的每一页是4096个字节，那么偏移量就用12个bit来表示(因为2^12=4096)，举个例子

虚拟内存地址0x00f11011，那么页编号是0x00f11 偏移量是011

这个时候CPU里面有一个表写着

0x00f11 --> 0x01010

注意CPU只是做页的映射，偏移量还是那个

所以我们从虚拟内存地址0x00f11011得到了物理内存地址0x01010011，这个时候就可以具体到某一个内存地址来读取数据了

多级分页表

由于每个进程都会有一套虚拟内存地址，所以每个进程都会有一个分页表，为了查询更块，分页表也会保存在内存中

分页表有多种实现方式

最简单的方法

我们前面讲的线性列表就是最简单的方式

但这有一个缺点

我们每个进程空间都会为栈和堆的增长预留了地址，但栈和堆很少能够占满进程空间

在这种情况下，没有用到的内存空间也具有映射关系，很多条目就会没有被真正用到(没有数据用啥用)

因此，Linux中的分页表，采用了多层的数据结构，能够有效减少分页表的空间

多层的数据结构

其实和上面的线性结构区别就在于，这里分多了几级，最新的Linux系统中的分页表多达3层

举一个双层的例子

虚拟内存地址还是0x00f11011


0x00 ---> 一级表 ---> 二级表(这个二级表所有开头都为0x00) ---> 查询f11映射 ---> 得到对应物理内存的页地址 ---> 根据偏移量 --> 得出最终地址

其实我们可以看到和上面的线性结构相比，双层结构只是变成了树状结构，3层也就变成了更大的树

多层分页表的基本原理都是一样的
