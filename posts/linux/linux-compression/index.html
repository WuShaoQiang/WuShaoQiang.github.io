Linux Compression

2019年7月21日
| Linux
| 阅读

Last day, we took a quick look on file system, I just want to give you guys some basic understanding about that, If you are a software developer(engineer), may be that’s enough for you.

Today, let’s talk about a very basic and useful tool in linux –> compression

Quick talk

Basically, we use compression all the time, including surfing the internet, the server will give client(browser mostly) the compressed data instead of raw data which may cause the performance slow.

Imagine that, if you have a lot files(small file mostly), and you want to copy all of them into a flash disk, you may find out the transmission speed is slower than the normal case(transmit a large file), so, compression can save you a lot time to transmit something fragmented

File suffix

We do know, filename is meaningless for linux which is totally different for windows. But filename(filename suffix) is meaningful for people to understand, so we use file suffix to identity what compression method is used to compact a file.

  • *.Z compress

  • *.gz gzip

  • *.bz2 *.bzip2

  • *.tar tar

  • *.tar.gz tar and gzip

  • *.tar.bz2 tar and bzip2

Notice: tar just collects data, it doesn’t compress anything. Usually, we will combine it with compression method like gzip and bzip2

BTW, compress is out of date now, so it won’t be necessary to talk about that.

gzip

gzip is one of the popular compression command. It can uncompress file which is compressed by gzip, zip or compress

Let’s compress a file

gzip -v filename    //it will show you the compression ratio

the command will output a file name filename.gz, but gzip will delete the origin file

if you want to keep the origin file, you can simply add a flagc,like:

gzip -c filename > filename.gz

If you just want to take a look at the compressed file, you can simply use

zcat filename.gz

You can tell from the name of this command that it does the same thing as cat.

Here, I will give you more options about compression.

gzip -[1-9]     //compression level, as the number increase
                //the compression operation will take more time
                //but the compression ratio will also go up(default is 6)

uncompress

It is easy to uncompress by using d flag

gzip -d filename.gz

Also, it will delete the *.gz file, if you don’t want to do it, you know how to do :-)

bzip2

bzip2 is an another powerful compression tool. Technically, it is better than gzip in compression ratio

Good for us, they are basically the same thing

So, we don’t need to remember a lot command

There is a little bit different between them

bzip2 -z filename   //zip the file need -d flag

Also, we can use bzcat to read the file content

tar

If you notice that all we talked above is just simply compress one file.

So, we need an another tool to gather all files we want to compress into one file and then we compress it.

tar is our hero.

tar can be a little bit complicated because of too many flag, you can check the man page yourself if you don’t know what exactly the command is.

I am gonna cover some basic usages

Remember the compression method we just talked about ==> gizp and bzip2

tar can easily use these compression method

-j      //use bzip2 (recommend)

-z      //use gzip

-c      //create a new tar file

So, simply use below command to compress files or directories

tar -[j|z]cv -f filename.tar.[gz|bz2]   [target files and directories]

//Example
tar -jvc -f picture.tar.bz2 /home/me/Picture

If we want to take a look at the content of compression file, simply use:

tar -[j|z]tv -f filename.tar.[bz2|gz]

//Example
tar -jtv -f picture.tar.bz2

OK, uncompress

tar -[j|z]xv -f filename.tar.[bz2|gz] -C target_path

//Example
tar -jxv -f picture.tar.bz2 -C ~/picture_test

It may be a little confusing, let me give you a tip to remember them easily

Tips

Usually, we will combine the tar and compression method

First step, you need to choose the compression method you want(or you need)

  • opts

    • -j bzip2

    • -z gzip

Usually, we want to see the files that have been included, so use -v

So, here we create a command like this

tar -jv
//OR
tar -zv

The second step is what you want to do, options:

  • compress: use -c to create compression file

  • uncompress: use -x to extract file, and use -C to locate the path you want to put files in

  • search/look: use -t to take a look at the file inside

So, we shouldn’t use two of them at the same time since they have totally different action

Now command is like that

tar -jvc        //create compression file
tar -jvt        //have a look
tar -jvx        //uncompress file

The final step is to target the file using -f filename.tar.[bz2|gz]

I hope this will be easy to remember

More options

-p      //keep the permission and property

-P      //keep absolute path

--exclude=FILE  //exclude specified file

In most cases, we don’t need to keep the absolute path by using -P

Thanks

I hope you will enjoy this topic, and learn some skill about linux compression
