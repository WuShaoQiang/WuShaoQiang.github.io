Linux Permission

2019年7月20日
| Linux
| 阅读

Today, I am gonna talk about linux permission which is very basic for linux user.

Basically, we have three different role in linux

Owner

Group

Other

It is easy to understand what they mean.

ls -al

In your home directory, type ls -al command and click Enter, you will see the output like:

-rw-------   1 user group  254 Oct 22  2018 .xsession-errors.old

Fields meaning

As we can see, this line include several field separated by white space

Let’s find out what they mean

-rw-------

The first field always has 10 characters

  • the first character means its file type

    • -: file
    • d: directory
    • l: link
    • c: serial port device
    • b: block

  • the second one is rw-, it presents what permissions the owner has, as we can see, the owner has read and write permission

  • the third one is ---, it presents what permissions the group has, we know that the group has no permission at all,

  • the last one is also ---, so Other has no permission to the file

Let’s see another example:

drwxrwxr-x

First, we know it is a directory, and its owner have rwxpermission

It is different between file permission and directory permission, we will talk about it later

its group has rwx permission and other has r-w permission

I hope you can understand the permission schema which is grouped by 10 characters

OK, let’s jump to the next field

1

It is the link number, and related to i-node

user group

these two fields is quite simple, they present the file owner and its group respectively

254

It presents the file size and the default unit is Byte

So, we know the file size is 254Bytes

Oct 22  2018

this is a date when the file is last modified

and the last one is filename with no doubt

Difference between file permission and directory permission

File permission is quite simple to understand by human

  • r: read file content

  • w: modify file content(not delete)

  • x: execute file

Directory permission will be a little be difficult to understand compare to the file permission

  • r: just read the filename inside the directory

  • w: This permission allow you to delete the items inside the directory no matter what’s the file permission you have. So basically we don’t easily give someone this permission (delete create rename file)

  • x: This permission allow you to use directory as work directory (like cd command), it can be use to delete directory

In general, file permission is related to the file content, but the directory is the controller of file existence and filename that something is kind of like file property

Now, I want to emphasize the directory permission

In directory permissions, x is the most complicate permission

Imagine, If you have the file read permission but you don’t have directory x permission. Can you read the file content?

The answer is NO

Why?

Because we don’t have the permission to work in the directory that we don’t have x permission.

Basically, if you want to read a file content, the first step you need if go inside the directory and then open the file, but you can’t go inside the directory so you won’t be able to read the file content? Does it make any sense to you? I hope so.

If you do not understand, you should google What is the work directory, and then come back

JUST REMEMBER

Directory just like a container, and files are items which are stored in the container

only when you have x permission of a container, you are allowed to open it. If you can’t open it? then nothing will go well with you

So, if you can open the container, and you have w permission, you can drop whatever you want in that container even you don’t have file permission, right? because drop the item doesn’t need the item content

What if r permission, ye, you can basically read the item’s name not the item’s content

Make sense?

chmod       //change permission

chgrp       //change group

chown       //change owner
