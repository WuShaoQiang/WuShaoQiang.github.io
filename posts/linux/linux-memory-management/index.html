Linux Memory Management

2019年10月29日
| Linux
| 阅读

这里主要说的是内核内存，而不是我们平时写代码时的用户内存

page

页(page)是内存管理的单元，虽然在物理内存中最小处理单位可能是字节，但从虚拟内存的角度来说，页就是最小单位

一般来说我们现在使用的电脑系统，页的大小为4KB

$ getconf PAGE_SIZE
4096

可以用上面的这个命令来查看当前物理机的页大小

相应的结构体

相应的结构体定义在了include/linux/mm_types.h

/*
 * Each physical page in the system has a struct page associated with
 * it to keep track of whatever it is we are using the page for at the
 * moment. Note that we have no way to track which tasks are using
 * a page, though if it is a pagecache page, rmap structures can tell us
 * who is mapping it.
 */
struct page {
	unsigned long flags;		/* Atomic flags, some possibly
					 * updated asynchronously */
	atomic_t _count;		/* Usage count, see below. */
	struct list_head lru;		/* Pageout list, eg. active_list
					 * protected by zone->lru_lock !
					 */
	/*
	 * On machines where all RAM is mapped into kernel address space,
	 * we can simply calculate the virtual address. On machines with
	 * highmem some memory is mapped into kernel virtual memory
	 * dynamically, so we need a place to store that address.
	 * Note that this field could be 16 bits on x86 ... ;)
	 *
	 * Architectures with slow multiplication can define
	 * WANT_PAGE_VIRTUAL in asm/page.h
	 */
#if defined(WANT_PAGE_VIRTUAL)
	void *virtual;			/* Kernel virtual address (NULL if
					   not kmapped, ie. highmem) */
#endif /* WANT_PAGE_VIRTUAL */
#ifdef CONFIG_WANT_PAGE_DEBUG_FLAGS
	unsigned long debug_flags;	/* Use atomic bitops on this */
#endif

#ifdef CONFIG_KMEMCHECK
	/*
	 * kmemcheck wants to track the status of each byte in a page; this
	 * is a pointer to such a status block. NULL if not tracked.
	 */
	void *shadow;
#endif
};

因为结构还是比较复杂，将复杂的union删除了，因此只是简单的说一些容易理解的字段

  • flags是表示这个page的状态，比如dirty，locked等

  • _count是表示该page被引用的次数

  • virtual在一般情况下就是我们所说的虚拟内存

我们从结构体的注释中可以知道

page结构是和物理页相关的，也就是说这种数据结构的目的在于描述物理内存本身，而不是描述所包含的数据

Linux内核通过这样的一个结构体来管理系统中所有的物理页，所以说，每一个物理页都要对应一个这样的结构体

zone

the kernel use the zones to group pages of similar properties

我们可以认为，在不同的域中的page将有着不同的作用

在硬件层面上，这些内存还是一样的，只是Linux为了解决一些硬件上的问题(比如有些硬件只能访问0~16M的内存)而抽象出来的zone

所以zone在物理层面上是没有隔离的

$ cat /proc/zoneinfo | grep Node
Node 0, zone      DMA
Node 0, zone    DMA32
Node 0, zone   Normal
Node 0, zone  Movable
Node 0, zone   Device

这是我电脑上的zone种类，其中这里没有HIGHMEM

在32位机系统上，DMA和DMA32是没有区别的。因为我的电脑是64位的，因此当有些硬件需要使用32位的DMA(Directly Memory Access)的时候，就会用到DMA32

Normal是最常见的内存，我们平时用到的也是这一种

Movable和Device分别是为了防止内存碎片化和支持设备热插拔的zone

还有我刚提到的HIGHMEM，这个一般是在32位机上出现，因为有可能真实的物理内存要比内核能够映射的虚拟内存还大，所以需要有这样的zone。但是因为64位机一般都不会有映射不够的情况，因此就没有这个zone(或者说变成了Normal)
