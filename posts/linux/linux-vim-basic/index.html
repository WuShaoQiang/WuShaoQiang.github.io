Linux Vim Basic

2019年7月21日
| Linux
| 阅读

Last time, we talked about linux compression

This time, I am gonna talk about basic usage about vim

vim is a highly recommended editor in UNIX-Like system, so it is necessary to learn that

To be honest

I am not one of the vim-fans. Mostly, I use other editor(like VScode) or IDE to develop software, I use vim only when I just want to edit a config file or just couple lines of code which is not worth opening a editor(or IDE)

But, vim is really important because other editor(or IDE) may not exist in some machines. At that time, you must know how to use vim to edit or do something else.

What can I tell you

I won’t describe something like how to open vim or how to exit vim, these questions are just too simple, please check by yourself

I am gonna talk about how to improve you performance at vim

Let’s go

In normal mode

Home End PgUp PgDn      //these button can be easy to understand for beginner

move cursor to the begin(end) of file

G       //end (we can simply type (shift+g))

gg      //begin

move to a specified line using [number]G

like 20G => move to 20th line

move to next(last) matched when you search in file

n       //next

N       //last

search

/word       // find next word

?word       // find last word

replace

:n1,n2s/word1/word2/g       //from line n1 to n2, replace word1 with word2

word1 can be regular expression

If you want to replace all, use $ instead of n2

g is global

you can also

  • i: ignore case

  • c: need confirm

delete line

  • dd: delete current line

  • ndd: delete n lines

undo redo

  • u: undo

  • [CTRL]+r: redo

In command mode

(un)set line number:

  • setnu :set

  • setnonu :unset

save as

:w filename

you can copy these table as a tip for yourself, when you forget the shortcut, just simply find in the tip.

A little tips

In vim, number is meaningful, it means how many times the operation will be operated.

like 50dd => delete 50 line

50u undo 50 times

So, if you want to do repeat action, you can try [number]+command combination, maybe you will be surprised
