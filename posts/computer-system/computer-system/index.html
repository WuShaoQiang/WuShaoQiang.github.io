第二章

溢出

整数注意溢出，浮点数注意大数吃小数(虽然也要注意溢出，但是大数吃小数情况更加容易出现)

大数吃小数的例子

func main() {
	a := 1000000000000000.0
	b := a
	c := 0.001

	fmt.Println((a - b) + c)    //不会吃
    fmt.Println(a - (b + c))    //会吃
}

性能与底层bit

二维数组，一行一行扫描比一列列扫描性能要好

无符号和有符号的数字需要小心分析它底层的bit，举例子

无符号：1 1 1 1 --> 2^3+2^2+2^1+2^0 = 15

有符号：1 1 1 1 --> -2^3+2^2+2^1+2^0 = -1

###大小端

大端法：最高有效字节在最前面的方式

比如0x01234567,在内存的表示是

01 23 45 67

小端法：最低有效字节在最前面的方式

比如0x01234567,在内存的表示是

67 45 23 01

一般操作系统用的都是小端法

兼容

即使是完全一样的进程，运行在不同的操作系统上也会有不同的编码规则，二进制编码是不兼容的

位操作

布尔代数

a^a=0

a^b^a = b

a&(b|c) = (a&b) | (a&c)

a|(b&c) = (a|b) & (a|c)

一个智力题：在不适用第三个空间的时候，如何交换两个数值(对性能没有什么影响)

func main() {
	a := 5
	b := 2
	swap(&a, &b)
	fmt.Println(a, b)
}

func swap(x, y *int) {
	*y = *x ^ *y
	*x = *x ^ *y
	*y = *x ^ *y
}

结果为2 5

通过异或即可达到这样的效果

位级运算的一个常见的用法就是实现掩码运算

比如x=0x87654321

  • 保留最低有效字节：x & 0xff

  • 除了最低有效字节，其它位都取补：0xffffff00 ^ x

  • 最低字节全部设成1，其它都不变：0x000000ff | x

注意逻辑运算与位运算符号的相似，但功能完全不同

移位操作

C语言标准中没有明确定义有符号类型右移是属于逻辑右移还是算术右移

Golang的移位操作和C的可能有所不同

有符号位的情况下

  • 右移：符号位会保持，也就是说原本是1的不会因为移位变成0

举个例子

-1 : 1111111....11(一共64bit)

-1 >> ? 其中问号表示任何正整数，这个结果都还是-1，可以验证golang的移位操作保持符号位
  • 左移：右边用0填补，左边溢出舍弃，在移位过程可能会丢失符号位

无符号位的情况下

  • 右移：左边补零

  • 左移：右边补零
