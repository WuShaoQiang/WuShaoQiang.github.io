题目

给定一个二叉搜索树（Binary Search Tree），把它转换成为累加树（Greater Tree)，使得每个节点的值是原来的节点值加上所有大于它的节点值之和。

例如：

输入: 二叉搜索树:
              5
            /   \
           2     13

输出: 转换为累加树:
             18
            /   \
          20     13

我的解答

这个题主要在于发现它的规律,节点所有比自身大的节点到底有哪些

用树的思想去做会比较麻烦

// 执行用时 : 540 ms, 在Convert BST to Greater Tree的Go提交中击败了35.56% 的用户
// 内存消耗 : 41.9 MB, 在Convert BST to Greater Tree的Go提交中击败了100.00% 的用户

func convertBST(root *TreeNode) *TreeNode {
	if root == nil {
		return nil
	}

	orderByIn(root, nil)

	return root
}

func orderByIn(root *TreeNode, lastNode *TreeNode) {

	if root == nil {
		return
	}

    // 表明是有转折点的
    // 有转折点的意思是这个节点在某个祖先节点的左子树上(所以要加上这个祖先节点)
	if lastNode != nil {
		root.Val += lastNode.Val + subtreeSum(root.Right)
	} else {
        // 如果没有转折点,只用加这个节点的右子树就可以
		root.Val += subtreeSum(root.Right)
	}

    // 右子树就继承父节点包含的lastNode(因为是在同一个左子树转折点)
	if root.Right != nil {
		orderByIn(root.Right, lastNode)
	}

    // 左子树的话就证明现在这个点就是下一个点的转折点
	if root.Left != nil {
		orderByIn(root.Left, root)
	}

}


// 简单的遍历所有子节点的数值并且相加
func subtreeSum(root *TreeNode) int {
	if root == nil {
		return 0
	}

	return root.Val + subtreeSum(root.Right) + subtreeSum(root.Left)
}

从树的角度来看,能够比一个节点大的只有

  • 它右子树的节点

  • 它祖先节点的转折点(比较难理解),比如一个节点在某个节点的左子树,那么就是这个祖先节点比它大

我们需要注意,如果用这种方式一定要在根节点开始

最优解答

从时间角度来看的最优解

这个时候就不是用树的思想了,而是将BST换成一个有序数组

// 执行用时 : 372 ms, 在Convert BST to Greater Tree的Go提交中击败了100.00% 的用户
// 内存消耗 : 72 MB, 在Convert BST to Greater Tree的Go提交中击败了67.74% 的用户

// 用反转中序遍历
func convertBST(root *TreeNode) *TreeNode {
	if root == nil {
		return nil
	}
	res := make([]*TreeNode, 0)
	orderByIn(root, &res)

	for i := 1; i < len(res); i++ {
		res[i].Val += res[i-1].Val
	}

	return root
}

// 这里用了反向的中序遍历
// 中序遍历是升序的
// 反向意味着降序
func orderByIn(root *TreeNode, res *[]*TreeNode) {
	if root.Right != nil {
		orderByIn(root.Right, res)
	}

	*res = append(*res, root)

	if root.Left != nil {
		orderByIn(root.Left, res)
	}
}

我们把BST看作一个降序的数组,最大的在前面,小的在后面

将这个问题转换为了,数组的小数加上全部大于它的数,这就很简单了

遍历一次相加就好了

难疑点&核心

501 530 538三题都应用到了BST转换有序数组,可以看出这类思想在解决问题还是很有用的
