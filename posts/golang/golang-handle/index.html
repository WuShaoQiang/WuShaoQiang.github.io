Golang Handle

2019年4月11日
| Golang
| 阅读

本文主要解释Golang中http-handler几个容易混淆的概念

HandleFunc

源代码

在源码文件(net/http/server.go)中定义了一个默认的路由管理器(Mux)，我们如果直接调用

http.HandleFunc(...)

其实就用了默认的路由管理器，然后通过这个默认的路由器来调用路由管理器的方法

当然我们也可以自己定义一个路由管理器，然后通过这个对象来调用这些方法,和调用默认Mux不同的是

myMux.HandleFunc(...)

来看一下源码的这两个函数，验证上面所说的。

// HandleFunc registers the handler function for the given pattern
// in the DefaultServeMux.
// The documentation for ServeMux explains how patterns are matched.
func HandleFunc(pattern string, handler func(ResponseWriter, *Request)) {
	DefaultServeMux.HandleFunc(pattern, handler)
}

利用DefaultServeMux来调用HandleFunc()方法

// HandleFunc registers the handler function for the given pattern.
func (mux *ServeMux) HandleFunc(pattern string, handler func(ResponseWriter, *Request)) {
	if handler == nil {
		panic("http: nil handler")
	}
	mux.Handle(pattern, HandlerFunc(handler))
}

HandleFunc方法实质上是调用了Handle方法

我们传入HandleFunc的参数是一个字符串和一个函数

HandlerFunc

在上面的代码中我们还能看到最后一行参数里面调用了HandlerFunc(handler)

// The HandlerFunc type is an adapter to allow the use of
// ordinary functions as HTTP handlers. If f is a function
// with the appropriate signature, HandlerFunc(f) is a
// Handler that calls f.
type HandlerFunc func(ResponseWriter, *Request)

这是将原本传入的函数，显示转换为HandlerFunc类型

因为Golang对类型是严格的，哪怕定义两种类型

HandlerFunc的一个方法如下

// ServeHTTP calls f(w, r).
func (f HandlerFunc) ServeHTTP(w ResponseWriter, r *Request) {
	f(w, r)
}

这个方法是为了符合Handler接口

type i1 int
type i2 int

Go都认为这两种类型是不同的，如果接口里面明确要求要某种类型，我们是要进行强制转换的

所以当Handle这个函数的第二个参数需要HandlerFunc类型的参数时，我们需要强制转换

Handle

// Handle registers the handler for the given pattern.
// If a handler already exists for pattern, Handle panics.
func (mux *ServeMux) Handle(pattern string, handler Handler) {
	mux.mu.Lock()
	defer mux.mu.Unlock()

	if pattern == "" {
		panic("http: invalid pattern")
	}
	if handler == nil {
		panic("http: nil handler")
	}
	if _, exist := mux.m[pattern]; exist {
		panic("http: multiple registrations for " + pattern)
	}

	if mux.m == nil {
		mux.m = make(map[string]muxEntry)
	}
	mux.m[pattern] = muxEntry{h: handler, pattern: pattern}

	if pattern[0] != '/' {
		mux.hosts = true
	}
}

这是最终的注册函数，参数传到这里之后会存储起来

Handler

type Handler interface {
	ServeHTTP(ResponseWriter, *Request)
}

这是一个接口，只要实现了接口函数，就可以被认为实现了该接口

我们在上面说到的HandlerFunc 就实现了该接口，因此当函数需要一个Handler接口的参数时，我们可以传入HandlerFunc

总结

整个转换过程是

我们自己写的func handler (w http.ResponseWriter,r *http.Request)

传入http.HandleFunc()

通过HandlerFunc将我们写的函数强制转换为HandlerFunc类型，然而HandlerFunc类型实现了Handler的接口

http.Handler()的第二个参数要求传入Handler接口的参数

我们将HandlerFunc传入http.Handle()

完成注册

一般情况下，我们在写后端服务的时候，用的是HandleFunc 这样只要我们写出的handler符合func (w http.ResponseWriter, r *http.Request)这个类型就可以注册

但是如果我们调用的函数返回的是一个Handler类型数据，我们就要用http.Handle()，这个第二参数才是传入Handler接口的

总的来说，要弄清楚接口类型以及参数类型，不可将不同类型的参数传入。
