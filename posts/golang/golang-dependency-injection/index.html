Golang 依赖注入

2019年4月26日
| Golang
| 阅读

面试时遇到一道题，问依赖注入带来了什么好处。。。？？？我连啥是依赖注入都不知道，真是打扰了，赶紧补回来

没有DI

把所有创建变量放在New函数里面

type Person struct{
    name string
    age  int
}

type Student struct{
    person *Person
}

func NewPerson() *Person{
    return &Person{}
}

func NewStudent() *Student{
    return &Student{
        person:&Person{},
    }
}

func main(){
    s := NewStudent()
}

我们不能按照自己的意愿传入想要的参数,这样main函数比较简单

有DI

type Person struct{
    name string
    age  int
}

type Student struct{
    person *Person
}

func NewPerson(name string,age int) *Person{
    return &Person{
        name:name,
        age:age,
    }
}

func NewStudent(person *Person) *Student{
    return &Student{
        person:person,
    }
}

func main(){
    p := NewPerson("abc",12)
    s := NewStudent(p)
}

这里我们就是有依赖注入，能够按照我们的需要传入参数，但是main的代码会随着结构体的传递变得很长

一个比较生动的例子

电脑的USB，可以接U盘，接硬盘等，所以我们来决定到底是什么，这就是依赖注入的模式

假设电脑的USB在初始化的时候就已经固定是某一个硬件了，那么我们就无法改变了

什么是依赖注入(依赖注入干了什么)

来自Bhavya Karia

transferring the task of creating the object to someone else and directly using the dependency is called dependency injection.

来自wiki

The client delegates the responsibility of providing its dependencies to external code (the injector).

This means the client code does not need to know about the injecting code, how to construct the services or even which actual services it is using

This separates the responsibilities of use and construction

就像上面的两端程序一样

如果结构体Student需要调用Person的New函数，那么这就是依赖

如果通过参数传递，那么就实现了依赖注入，但是这些注入会让程序变得很长，以至于难以阅读
