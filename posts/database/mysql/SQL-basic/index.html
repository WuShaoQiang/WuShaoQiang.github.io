SQL 基础

2019年4月22日
| 数据库
| 阅读

笔者之前学了一些基础的Mysql的语法，现在进一步学习

关系型数据库用的语言标准就是SQL，只是不同的数据库有自己的拓展，所以学SQL标准可以说是一举多得吧

非关系型数据库有自己的一套语言

只针对Mysql的SQL语句

SELECT

选定要输出的列

SELECT column_name,column_name FROM table_name;

将所有列输出

SELECT * FROM table_name;

SELECT DISTINCT

如果一个列中包含了很多相同的，比如国家里面有很多人是中国的、日本的等等，我们指向查看有多少种的话，这样看起来很费劲

介绍一个解决这种问题的命令

和SELECT语法基本一样，只是这里不会输出相同的字段

SELECT DISTINCT column_name,column_name FROM table_name;

WHERE

WHERE子句用于提取那些满足指定标准的记录

SELECT column_name,column_name FROM table_name WHERE column_name operator value;

例如，我要id大于3的user的id和name字段

SELECT id,name FROM user WHERE id > 3

这里提醒一点，当实例是文本字段需要加单引号，如果是数值字段，就像上面一样直接写就好

WHERE的运算符

WHERE column_name operator value

这里的operator就是运算符的位置

运算符有：

  • =,>,>=,<,<= 这些就比较好理解了

  • <> 不等于

  • BETWEEN 在某个范围，用法WHERE column_name BETWEEN value1 AND value2

  • LIKE 搜索某种模式

  • IN 多个可能值，用法WHERE column_name IN (value1,value2,...)，这种也可以和OR替换

  • AND 同时满足两个条件

  • OR 满足一个即可

  • NOT 非

  • is null

运算优先级：() not and or

对于LIKE

  • % 表示多个字值，_ 下划线表示一个字符；

  • M% : 为能配符，正则表达式，表示的意思为模糊查询信息为 M 开头的。

  • %M% : 表示查询包含M的所有内容。

  • %M_ : 表示查询以M在倒数第二位的所有内容。

ORDER BY

这个是为了排序的

  • ASC升序

  • DESC降序

例子

SELECT column_name,column_name FROM table_name ORDER BY column_name,column_name ASC|DESC;

INSERT INTO

INSERT INTO table_name VALUES (value1,value2,value3，...);

或者

INSERT INTO table_name (column1,column2,column3,...)VALUES (value1,value2,value3,...);

UPDATE

UPDATE table_name
SET column1=value1,column2=value2,...
WHERE some_column=some_value;

用于更新现有的数据，用WHERE子句筛选

这里要注意，如果UPDATE不加WHERE子句，会将table里面所有的数据都更新

可以通过set sql_safe_updates=1;来保证UPDATE语句一定要加WHERE和LIMIT

DELETE

DELETE FROM table_name
WHERE some_column=some_value;

和UPDATE类似，这也是对已有数据进行处理

如果我们不小心忽略了WHERE子句，会把table里所有数据删除掉

同样的，设置sql_safe_updates=1;可以保护这种意外情况

LIMIT

SELECT column_name
FROM table_name
LIMIT number;

这个规定要返回的记录数目

AS(别名)

列的别名

SELECT column_name AS alias_name
FROM table_name;

表的别名

SELECT column)name FROM table_name AS alias_name;
