Mongodb Replica Set Deployment

2019年5月30日
| 数据库
| 阅读

Before start

official document

Today, I worked in company program, using mongodb replica set to specify multiple database servers. And this article is for introducing How to deploy a replica set in our own PC for testing and development

It is almost the same for deploying on different host which it used for production

Required

I assume you already know how to use mongodb(e.g. mongod,mongo etc.)

The version of my mongodb is 3.2

Step

I will cover all the steps below

Start mongod

I will start mongod server in two different ways

One(just an example)

mongod --bind_ip localhost --port 20001 --dbpath ~/mongo/db --logpath ~/mongo/log/mongo.log --fork --replSet rs0
  • you can check what are the fields for by executing mongod -h

Another

mongod -f ~/mongo/mongo.conf

Both of them are just examples, you need to modify some configuration base on what you want.

Start more than one mongod

I just started only one mongod, but a replica set need more than one databases, simply, we just need to run the above command line

!!Warning!!

  1. You should use different port for each mongod

  2. You should use the same replSet argument(above:rs0)

Start client

We have started a replica set of mongod

Now, we need a client(application or mongo)

Use mongo as client

Remember, your first mongod will be primary DB server, and the rest will be secondary, so there is a master-slave relation

If we use mongo client, we need to connect to the primary DB so that we can do read&write operations

mongo --port 20001

PS: You can set the permission of secondary DB, but that is not the topic in this article

Use your application

If you are a developer, you probably use it in your application

For example, I am using Go

There is a 3rd package used for supporting mongodb called go-mongodb-driver in Github

So I just put these codes in my app

options.Client().SetHost([]string{"host1","host2","host3"})

The driver will do the complicate job for me. It is really helpful

Important things

  • Do use different replica set name if you just want them in the same set

  • More setting can be accessed in mongodb.org
