Mongodb Installation And Config

2019年5月11日
| 数据库
| 阅读

总结一下最近在配置MongoDB时遇到的坑吧

首先,在搜索安装教程的时候,很多文章写的是直接安装.tar包,然后解压缩来用

嗯,确实是个直接的方法,用起来也可以,只是稍微麻烦自己去设置环境而已

直接下载tar包

解压后,要将/bin添加到环境变量

然后还要自己创建一个mongod.conf,往里面配置一些东西.或者可以直接在每次开启mongod的时候加入参数(比较麻烦)

这里是我初步的一个配置文件

# mongod.conf

# for documentation of all options, see:
#   http://docs.mongodb.org/manual/reference/configuration-options/

# Where and how to store data.
storage:
  dbPath: /home/shelljo/mongodb/data/db
  journal:
    enabled: true
#  engine:
#  mmapv1:
#  wiredTiger:

# where to write logging data.
systemLog:
  destination: file
  logAppend: true
  path: /home/shelljo/mongodb/log/mongod.log

# network interfaces
net:
  port: 27017
  bindIp: 0.0.0.0


# how the process runs
processManagement:
  timeZoneInfo: /usr/share/zoneinfo

security:
  authorization: enabled
#operationProfiling:

#replication:

#sharding:

## Enterprise-Only Options:

#auditLog:

#snmp:

其实这里面还遇到一些别的坑,不太记得了,我搜索了几篇关于安装的教程都是这个样子

官网教程

其实一切都能变得很简单…

官网教程

按照官网教程其实真的不用一分钟就把环境呀,默认配置文件呀什么的搞好了.

具体就是如下命令了(建议直接看官网,万一它更新了呢)

sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv 9DA31620334BD75D9DCB49F368818C72E52529D4
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu xenial/mongodb-org/4.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-4.0.list
sudo apt-get update
sudo apt-get install -y mongodb-org

这几条命令就搞完了

官网安装的一些我认为的坑

官网后面介绍如何开启和关闭mongod服务,在我安装后(尝试了两三次重装)都没有

去/etc/init.d/mongod也找不到相关文件

Google了一下Copy别人的mongod也愣是不行,所以先放在一边了

这里我就按照比较麻烦的方式开启和关闭服务了

//开启
mongod -f /etc/mongod.conf //注意这里换成自己config文件的位置(默认是这个位置的)

//关闭
关闭要进入到mongo shell里面执行use admin然后执行db.shutdownServer()

因为我对脚本编写还不太熟悉,到后面再折腾如何将它注册进linux的服务上吧

一些配置

官网配置详解

其实基本配置有点基础的都知道怎么改,将一些文件定向到你想定向的地方

主要讲讲authrization

建议先看完这篇文章再看完关于MongoDB-role

这是一个认证,也就是说MongoDB里面可以有账户也就是role

具体参考上面链接

如果authrization没有设置

那么我们可以在MongoDB任意操作,加了这个就可以让数据库有了分层

比如说某某用户(某某公司员工)只是对某个数据库有读的权利(role:“read”),这个权利表明它只对这个数据库可见,并且只能读.

比如某某用户是大哥(role:“root”),那么它就可以对所有数据库可读可写可删除等等

再讲讲 bindIp

在我的设置里设置为0.0.0.0是因为我要在我的笔记本远程到这个数据库,如果是本机就设置为本机IP

提醒一下,如果设置为0.0.0.0并且对外开放了端口,并且还不设置authrization,那么所有人都可以连接到你的数据库,做任何事情了,所以记得要设置好authorization再开放出去
