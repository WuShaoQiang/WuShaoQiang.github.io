Golang goquery

2019年4月8日
| Golang
| 阅读

goquery是一个开源的库，针对解析HTML

今天在做爬虫的时候用到了一些功能，记录下来方便查阅

创建Document对象

    doc, err := goquery.NewDocumentFromReader(resp.Body)
    if err != nil {
        log.Fatalf("go query Error %s\n", err)
    }

doc对象就包含了我们要解析的HTML主体

一般我们这样用，Find()里面的字符串就是我们要填写的规则

    doc.Find("div").Each(func(i int, s *goquery.Selection) {
        fmt.Println(i,s.Text())
    })

规则

我们可以根据HTML的源码分析对应的属性id，class等等

  • div#id #id是某个div的id属性，筛选出此id的div标签

  • div.class .class 同理，是class属性

  • div[class],div[lang] 这个也是属性的筛选

  • div>p div嵌套的p标签，这里规定只有一级标签，一级以后的p标签不会被筛选出来

  • div p div嵌套的p标签，这里无论第几级都会被筛选出来

属性可以和嵌套一起使用，例如div.class>p>li

Size

num := doc.Find("ul>li>div.job-primary").Size()

取出标签内的属性

doc.Find("ul>li>div.job-primary>div>h3>a").Each(func(i int, s *goquery.Selection) {
    detail := s.Get(0).Attr[0].Val
    // fmt.Println(i, nextPage)
    jobs[i].Detail = detail
})

我在这里踩过一个坑

doc.Find("ul>li>div.job-primary>div>h3>a").Each(func(i int, s *goquery.Selection) {
    detail := s.Get(i).Attr[0].Val
    // fmt.Println(i, nextPage)
    jobs[i].Detail = detail
})

我把第二行的0改成了i就发生了异常

这里做下解释

在页面中，符合上面规则的a标签，是h3标签的一级标签，而且我们规定了只取一级标签(>),在HTML源码中符合一级标签的只有一个a标签，所以我们Get第一个就可以了，所以Get(0)是对的

Get(i)是毫无意义的，i在这里是指h3标签的数量(就是符合上述条件的地方个数)

那如果h3标签下的一级a标签不止一个呢？

我们可以这样获得a标签个数

len(s.Nodes)

然后就可以在内部遍历
