Golang Interface

2019年4月20日
| Golang
| 阅读

在说接口之前，我们先说一下类型，类型在Go语言里面可以大致分为

  • concrete type(具体类型,int,string…)

  • abstract type(抽象类型 io.Writer,io.Reader…)

具体类型我们知道它到底长什么样子，比如说int64，我们知道它在底层有8个字节

抽象类型我们不知道或者说我们不关注它长什么样，但我们知道它的用处，比如io.Writer是用来写的

具体化谈Interface

举个非常简单的例子，我们在读文件的时候，可能会这样做

func main() {
	f, err := os.Open("abc.txt")
	if err != nil {
		fmt.Println(err)
	}

	byte, _ := ioutil.ReadAll(f)
	fmt.Println(string(byte))
}

看一下ioutil.ReadAll()

func ReadAll(r io.Reader) ([]byte, error) {
	return readAll(r, bytes.MinRead)
}

这里需要传入的是一个io.Reader接口，为什么*File类型传入一样可以呢

这里io.Reader是一个接口，接口里面只要一个函数

type Reader interface {
	Read(p []byte) (n int, err error)
}

而*File里面也实现了这个接口的函数

func (f *File) Read(b []byte) (n int, err error) {
	if err := f.checkValid("read"); err != nil {
		return 0, err
	}
	n, e := f.read(b)
	return n, f.wrapErr("read", e)
}

所以说，实质上是执行了*File的Read函数，这就实现了多态，只要是实现了接口的函数，就可以成为合法的接口参数传递

抽象化谈Interface

接口实质上是定义一个交流规则，或者说是一个协议

举个生活中的例子

火车运集装箱，集装箱的规格是限定的，火车要符合集装箱的拉运规则才能拉集装箱(火车要实现了接口)

现在有一艘船也想拉集装箱，那么它只要实现了集装箱的规格就可以了(轮船实现接口)

所以我们不管交通工具是什么，只要是符合规则的(实现了接口函数)，那么它就是OK的

为什么用接口

  • writing generic algorithms

  • hiding implementation detail

  • providing interception points

这个特性能让我们的接口包分割

Interface组合

组合从概念上理解，就是简单的将接口拼凑一起，实现两个接口或者更多接口

io.ReadWriter就是个很好的例子，这个接口实现了Reader和Writer的接口，所以它能符合两种单独的接口参数

Interface大小

Interface应该越小越好，越小抽象化越高

The bigger the interface, the weaker the abstraction –Rob Pike

准则

  • interface{} says nothing

  • Be conservative in what you send, be liberal in what you accept

这里的send是指函数返回一个接口类型的时候要保守点，比如New() interface{}就要小心了，我们更加需要一个确定的类型，比如os.Open()(*File,err)，返回File类型是比较好的

如果是函数接收一个接口，可以自由点。比如io.Copy()就是接收接口的

  • Return concrete types receive interfaces as parameters

assertion

有两种方式

将抽象化的东西具体化

select t:=v.(type){
    case int:
    ...
    case string:
    ...
}

抽象化保留

func do(v interfaceP{}){
    s := v.(fmt.Stringer) //might panic if not the type

    s ,ok := v.(fmt.Stringer)  
}

推荐抽象化保留，这样更加容易
